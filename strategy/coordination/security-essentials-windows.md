## A. Защита от потери данных (Windows-специфика) — КРИТИЧНО

> Первоисточник — пост-мортем: сгенерированная команда из-за коллапса escape-символов между **zsh → tmux → PowerShell** снесла установку Windows. У тебя ровно этот риск: смешение Git Bash + PowerShell + автономные прогоны.

**Жёсткие правила:**
1. **Перед необратимой командой — эхо, потом исполнение.** Деструктив (`Remove-Item`, `rm -rf`, `git reset --hard`, `DROP`, `TRUNCATE`) сначала вывести как текст и подтвердить, лишь затем запускать.
2. **`Remove-Item -WhatIf` / `rm` с предпросмотром.** Для PowerShell — сперва `-WhatIf` (показывает что удалится, не удаляя), затем без него.
3. **Никаких относительных путей в деструктиве.** Только абсолютные. `rm -rf $VAR/` где `$VAR` пуст = удаление от корня. Проверять, что переменная не пустая.
4. **Физический бэкап отдельно.** Перед массовыми операциями — копия на другой диск/носитель, не в той же папке (которую снесёт та же команда).
5. **Не доверять кросс-shell escaping.** Не строить команды, которые проходят zsh→tmux→PowerShell. Один интерпретатор на команду. Многострочное — через heredoc, не через цепочку экранирований.
6. **Автономный режим (auto-accept) + права на запись = повышенный риск.** До выдачи пайплайну прав на запись — прогнать его в dry-run/sandbox.

## B. Хуки-гардрейлы (выборка из 20, под твои задачи)

> Ставить выборочно как PreToolUse/PreWrite-хуки. НЕ все 20 — только релевантные. Не дублируют Fablize stop-gate (тот про завершение, эти про безопасность).

| Хук | Что ловит | Зачем тебе |
|---|---|---|
| **SQL-инъекции** | конкатенация строк в SQL вместо параметров | FastAPI/Celery + БД в проектах — прямой риск |
| **Обработка секретов** | хардкод ключей/токенов в коде, лог секретов | биржевые ключи `trading_bots`, API-ключи лидген |
| **Test-gate (Stop)** | объявление «готово» при непройденных/незапущенных тестах | усиливает Fablize-гейт на тестах |
| **Pre-write линтер** | синтаксис/типы до записи файла | меньше сломанных коммитов |
| **Риск-команды в шелле** | `rm -rf /`, `chmod 777`, `curl \| sh` | смыкается с разделом A |
| **Ослабление auth** | отключение проверок (verify=False, CORS *, eval) | security-критичные пайплайны |
| **Broken localhost-fetch в контейнерах** | fetch на localhost из контейнера (не достучится) | если гоняешь агентов в Docker/VPS |
| **XSS** | unescaped вывод в HTML/шаблоны | веб-лендинги лидген |

**Не брать** (не твой кейс): специфичные для конкретных фреймворков, которых у тебя нет.

**Где взять реализацию:** хуки — это скрипты в `~/.claude/hooks/` + регистрация в `settings.json` (`PreToolUse`/`Stop`). У тебя уже есть инфраструктура хуков (`context-mode-cache-heal.mjs`, `nexus-session-start.ps1`) — добавлять по тому же шаблону. Конкретные реализации — через скилл `/update-config`.
