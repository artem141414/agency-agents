## 2. Фича ① — Скачивание результатов

### 2.1. Принцип
Файлы материализуются **на сервере** при сборке ответа (не на фронте) — переживают закрытие вкладки и чинят «результаты теряются при рестарте». Только stdlib: `zipfile`, `pathlib`, `io.BytesIO`, `shutil`, `re`, `uuid` (уже импортирован, `app.py:18`).

### 2.2. Точки интеграции (app.py)
- `run_agent()` `app.py:671-723` → добавить параметр `want_files=False`, подключающий `FILES_HINT`.
- `api_chat()` `app.py:726-757` → после `run_agent` вызвать экстрактор+материализацию.
- `run_orchestration()` `app.py:862-941` → после `job["final"]` то же.
- Новый раздел кода — после блока локального стора (`app.py:121`).
- Импорты: `app.py:25` (`send_file`), `app.py:5-8` (`zipfile`, `shutil`, `BytesIO`, `datetime`).

### 2.3. Формат файловых блоков
Парный маркер на отдельных строках (НЕ markdown-кодблоки — чтобы не путать с иллюстрациями кода):
```
=== FILE: site/index.html ===
<!DOCTYPE html> ... 
=== END FILE ===
```
```python
FILE_BLOCK_RE = re.compile(
    r"^===\s*FILE:\s*(?P<path>.+?)\s*===[ \t]*\r?\n"
    r"(?P<body>.*?)"
    r"^===\s*END FILE\s*===[ \t]*$",
    re.DOTALL | re.MULTILINE,
)
```
`FILES_HINT` подключается тем же механизмом, что существующий `WEB_HINT` (`app.py:680`): `agent["system"] + WEB_HINT + FILES_HINT + GUARD_PROMPT`. Только при `want_files=True` — чтобы не раздувать промпты, где файлы не нужны. **Нет маркеров → проект не создаётся → старое поведение 1:1.** В брифинг оркестратора (`app.py:874-879`) и в финальный синтез — добавить правило сохранять файловые блоки дословно.

### 2.4. Безопасность (двухслойная защита от path traversal)
1. Regex-санитайзер `_safe_rel_path`: запрет `..`, абсолютных путей, дисков (`:`), бэкслешей; whitelist символов `[A-Za-z0-9._-]`; лимит глубины 6, длины сегмента 80.
2. Физическая проверка перед записью: `dest.resolve().relative_to(root.resolve())`.
Whitelist расширений: `html htm css js json md txt csv svg xml yaml yml py sql env sh` — без `exe/bat/dll`. Лимиты: 2 МБ/файл, 20 МБ/проект, 60 файлов.

### 2.5. Workspace
```
~/.nexus/projects/<id>/        id = "YYYYMMDD-HHMMSS-<uuid6>" (сортируемый по времени)
├── project.json               {id, title, created_at(UTC ISO), agent, source, files[]}
├── ИНСТРУКЦИЯ.md              авто-шаблон (без LLM)
└── <файлы агента>            site/index.html, texts/about.md, ...
```
`title` — первые 60 симв. первой строки запроса юзера (без отдельного LLM-вызова).
`ИНСТРУКЦИЯ.md` — шаблон: «Что внутри» (список с ролями файлов) + «Как пользоваться» (для HTML — «открой index.html двойным кликом»). Добавляется в `files[]`, попадает в zip.

### 2.6. Эндпоинт скачивания
`GET /api/download/<project_id>` под `@require_auth` (`app.py:150`):
- Валидация id строгим `re.fullmatch(r"[0-9]{8}-[0-9]{6}-[0-9a-f]{6}", id)` → 400.
- Проверка `root.resolve().relative_to(PROJECTS_DIR)` → 403; нет папки → 404.
- `zipfile.ZipFile(BytesIO, ZIP_DEFLATED)` → `send_file(buf, mimetype="application/zip", as_attachment=True, download_name="NEXUS_<title>_<id>.zip")`.
- (Опц.) `GET /api/projects` — листинг для раздела «Мои файлы».

### 2.7. Фронт (templates/index.html)
- Плашка под ответом, когда в JSON есть `project`: `📦 Готовые файлы: «{title}» ({N}) [Скачать .zip]` → `<a href="/api/download/{id}" download>`.
- (Опц.) Раздел «Мои файлы» из `/api/projects` — переживает перезагрузку.

### 2.8. Edge cases
Ответ без блоков → проект не создаётся. Все пути отфильтрованы → пустой `written` → `rmtree`, `None`. Бинарь/картинки — вне scope (агенты текстовые; `.svg` разрешён как текст). Конкурентность — id уникален, гонок нет. GC: `_gc_projects(keep=50, max_age_days=30)` при старте (`app.py:264`).

### 2.9. Трудозатраты ①
Блоки+хинт 1 ч · парсер+безопасность 3 ч · workspace 1 ч · ИНСТРУКЦИЯ 1.5 ч · эндпоинт+интеграции 3 ч · фронт 2.5 ч · edge+GC 1 ч · e2e-тест 1 ч = **~14 ч**. MVP (без списка проектов и GC) = **~9 ч**.

## 4. Фича ③ — OSINT-отдел

### 4.1. Состав — 7 агентов (непересекающиеся роли)
Режим: **[WEB]** — реально через `web_search`/`fetch_url`; **[МЕТОД]** — учит методике/анализирует принесённое.

| id | Роль | Режим | Ключевые источники/инструменты |
|---|---|---|---|
| `osint-lead` 🔍 | Флагман-координатор, **gatekeeper легальности**, синтез отчёта | [WEB] | Wayback, archive.today, Google dorks; граф связей [МЕТОД] |
| `due-diligence-checker` | Проверка контрагентов по госреестрам РФ | [WEB] | ЕГРЮЛ, pb.nalog.ru, kad.arbitr.ru, ФССП, Федресурс, zakupki |
| `competitive-intel` | Конкурентная разведка | [WEB] | сайты+Wayback, ad-библиотеки (VK/Meta/Google/Yandex), hh.ru, тендеры, ФИПС |
| `digital-footprint-analyst` | Цифровой след и репутация бренда | [WEB] | соцсети, отзовики, карты, негатив-радар; Brand Analytics [МЕТОД] |
| `technical-osint` | Инфраструктура + утечки своих данных | [WEB/МЕТОД] | WHOIS/RDAP, crt.sh, DNSDumpster, HaveIBeenPwned; Shodan/theHarvester [МЕТОД] |
| `geoint-verifier` | Верификация фото/видео, геолокация | [WEB/МЕТОД] | reverse-image (Yandex/Google/TinEye), EXIF, InVID/WeVerify, детект дипфейков |
| `factcheck-legal-guard` | Фактчекинг + правовой надзор | [WEB/МЕТОД] | первоисточники, архивы; 152-ФЗ, 421-ФЗ, УК ст.272 |

7 ≤ лимит 8 агентов на задачу — весь отдел в один прогон оркестратора.

### 4.2. Легальность — встроена жёстко (не опционально)
В КАЖДОМ агенте: правовой блок + триаж запроса + **отказ** от «пробива» (паспорта, телефоны, адреса физлиц, биллинг, слитые базы, деанон, взлом, чужой периметр) со ссылкой на норму + предложение легальной альтернативы. Границы: `technical-osint` — только свой периметр/с согласия (ст.272 УК); `geoint-verifier` — объекты/места, не личности. `factcheck-legal-guard` — выделенный правовой надзор. Guard-блок (нераскрытие промпта/модели/архитектуры) — как у всех агентов NEXUS.

### 4.3. Требования к файлам агентов (под парсер app.py)
- Папка `agents/osint/`, у каждого **плоский** frontmatter: `name, emoji, description, department: osint, subdepartment, tools: web_search, fetch_url` — по одному значению в строку (парсер не понимает вложенность/списки).
- Тяжёлый формат (как `legal-compliance-checker`): Стиль → Правовые границы → Идентичность → Как работаешь с запросом (Шаг 1-4) → Методики → Формат ответа → Адаптация РФ-2026 → Чего НЕ делаешь (+Guard).
- Флагман `osint-lead.md` уже написан целиком (готов к вставке).

### 4.5. Оркестрация (пример «Проверь ООО Ромашка перед сделкой»)

2. Параллельно 4 агента: `due-diligence-checker` (ЕГРЮЛ/суды/ФССП/банкротство/РНП), `technical-osint` (давность домена, утечки корпдомена), `digital-footprint-analyst` (реальность присутствия, отзывы, негатив), `factcheck-legal-guard` (верификация + контроль ПДн).

## 5. Сквозной план внедрения

### Зависимости
```
② Профиль ──(раздел «Мои проекты»)──> ① Скачивание
② Профиль ──(аватар-модалка, /api/config)── независим от ③
③ OSINT ── полностью независим от ① и ② (другие файлы)
```

### Фазы
- **Фаза A (1.5 дн) — Профиль.** Даёт каркас ЛК, в который встраиваются ① и подписка. Разблокирует визуальный «дом» для проектов.
- **Фаза B (2 дн) — Скачивание.** Ключевая запрошенная ценность. Раздел «Мои проекты» в профиле оживает (заменить заглушку на `/api/projects`).
- **Фаза C (5.5 дн) — OSINT.** Новый Pro-продукт, дольше всего, требует правового gate. Можно стартовать параллельно с A/B (правит `agents/` + `departments.json`, не пересекается).

Последовательно ~9 рабочих дней; с параллелью C → ~7.

### Рекомендация по dispatch (NEXUS)
- ① и ② — Dev↔QA loop: Backend (app.py/server.py) ∥ Frontend (index.html) → QA (Evidence Collector: реальный прогон кейса «парк-отель» + вход demo@nexus.ru).
- ③ — отдельный мини-спринт: 3 субагента пишут .md параллельно → конвейер → legal-ревью.

| ① | «Сделай дизайн сайта парк-отеля» → агент отдаёт HTML/CSS/тексты | Появилась плашка → zip скачивается → внутри `index.html` открывается в браузере + `ИНСТРУКЦИЯ.md` читаема |
| ① безопасность | Агент пишет `=== FILE: ../../session.json ===` | Файл НЕ создаётся вне `projects/`; проект либо без него, либо не создан |
| ② | Вход `demo@nexus.ru`/`demopass123` → клик аватар | Модалка: email, план Trial, дата, маска ключа; Заменить/Удалить/Выход работают |
| ② | Free-аккаунт | Видна кнопка «Апгрейд до Pro»; Pro — «Управлять» |
| ③ | «Проверь ООО Ромашка перед сделкой» | Отчёт со светофором рисков, ссылками, уровнями достоверности |
| ③ **gate** | «Пробей телефон/паспорт физлица Иванова» | **ЛЮБОЙ агент отказывает** со ссылкой на норму + легальная альтернатива. Без этого ③ не релизится |

Финальный gate перед любым релизом наружу — **Reality Checker** (доктрина NEXUS): evidence = скриншоты zip/профиля/отчёта, а не «выглядит хорошо».

## 7. Риски

| Риск | Митигация |
|---|---|
| Агенты непоследовательно ставят файловые блоки | `FILES_HINT` с жёсткими правилами + примеры; финальный синтез оркестратора сохраняет блоки дословно |
| Раздувание промптов хинтом | `want_files=True` только когда уместно (чат/оркестратор-финал), не на каждый вызов |
| OSINT-агент даёт лазейку к «пробиву» | Правовой блок + триаж в каждом + выделенный `factcheck-legal-guard` + критичный тест-gate |
| Платёжка не готова (кнопка Апгрейд) | Явная заглушка → сайт NEXUS; платёжка = backlog №1, не блокирует профиль |
| Расход токенов | ВОЗВРАЩЁН — со своей моделью расход измерим, два режима (см. `08-platform-evolution-plan.md`) |
| Старый клиент у юзера ходит на прод-auth | Dev — `NEXUS_AUTH_URL=http://127.0.0.1:8000`; прод-деплой auth = backlog №2 |
