## Главный вывод: порядок запуска

**Плагин → SaaS → Self-hosted.** Не параллельно — последовательно, с переиспользованием.

Почему именно так:
1. **Плагин почти готов** — агенты уже в `.md`. Продукт = build-скрипт + лендинг + платёжка. Дни-недели.
2. **Токены платит клиент** (его API-ключ) → нулевой риск убытка, высокая маржа, не нужен token-metering.
3. **SaaS требует ядра** — auth + БД + биллинг + учёт токенов (4–8 недель), и там есть финансовый риск (см. ниже).
4. **Self-hosted = SaaS-ядро в single-tenant режиме** — переиспользует 90% работы по SaaS, поэтому идёт третьим.

## Три критических риска (выявлены агентами, требуют решения ДО старта)

### 🔴 1. Баг учёта токенов в прототипе (блокер SaaS)
`marketing-office/app.py:537-541` читает `response.usage` **только последней итерации** tool-loop. Агенты-разведчики делают много tool-вызовов → токены промежуточных итераций теряются. В SaaS это прямая **дыра в монетизации**: мы платим Anthropic больше, чем выставляем клиенту. Чинится `UsageAccumulator` (шаг 4 плана рефакторинга в арт.02). Для плагина неважно (токены клиента) — ещё один довод за «плагин первым».

### 🔴 2. У плагина нет DRM (стратегическое напряжение)
Плагин = git-репо, клонируется к клиенту в plaintext. **Все system-промпты — наш главный актив — оказываются на диске клиента.** Отзыв подписки блокирует только обновления, не уже скачанное. Вывод: плагин запускаем первым ради скорости и виральности, но **ключевую ценность выносим на наш MCP-сервер** (по образцу toprank/context-mode), а не держим целиком в `.md`. Дешёвый тариф плагина = «пробник», апселл в SaaS/Enterprise.

### 🔴 3. Anthropic API недоступен с РФ IP (блокер SaaS-хостинга)
Если SaaS-сервер в ДЦ РФ — вызовы `api.anthropic.com` падают. Решение: egress-relay (микро-VPS в NL/PL, ~€4–8/мес) через `ANTHROPIC_BASE_URL`. Ключ остаётся наш, ToS не нарушается.

## Экономика (из арт.01)
- Типовой диалог: ~$0.04 (input 3000т×$3/1M + output 2000т×$15/1M). С prompt caching — $0.02–0.03.
- **Pro $19/мес**: маржа ~18% без кэша → **40–60% с кэшем**. Prompt caching обязателен.
- **Team $79 при лимите 5000 сообщений — убыточен** ($250 токенов). Чинить: пул ~2000 + докупка.
- **Enterprise — только cost-plus**, не фикс с большим лимитом.
- Порог окупаемости SaaS: ~700–1000 платящих Pro. Плагин/self-hosted закрывают быстрее (высокая маржа).
