## 0. TL;DR

**Позиционирование (по строке на продукт):**
- **A. SaaS NEXUS** — AI-маркетинговая команда под Рунет в браузере, без своего API-ключа. Аудитория: SMB/маркетологи РФ.
- **B. Плагин Claude Code** — 150+ готовых профи-агентов в IDE за цену обеда, токены клиента. Аудитория: разработчики.
- **C. Self-hosted** — NEXUS внутри периметра клиента, поддержка + SLA. Аудитория: средний/крупный бизнес.

**Что первым:** **Плагин (B)** — почти нет разработки (агенты уже `.md`), высокая маржа (токены платит клиент), нулевой инфра-риск, виральный плацдарм. SaaS — второй (нужны auth+БД+биллинг+учёт токенов, 4–8 нед). Self-hosted — третий, по входящим Enterprise.

**Главные цифры:**
- Себестоимость типового диалога ≈ **$0.04–0.05 (≈4₽)**; с prompt caching — **$0.02–0.03**.
- **Pro SaaS $19/мес: маржа ~18% (риск) → 40–60% с кэшем токенов.**
- **Team $79 на полном лимите 5000 — убыточен**; чинится снижением пула до ~2000 + докупкой.
- **Enterprise — только cost-plus**, не фикс с большим лимитом.
- Плагин/self-hosted — высокая маржа, закрывают порог окупаемости платформы (~700–1000 Pro) быстрее.

**Биллинг:** Stripe/Paddle (мир) + ЮKassa (РФ). SaaS — гибрид seat+usage+кредиты; Плагин — seat-подписка+Lifetime; Self-hosted — setup-fee + абонплата + cost-plus.

## 1. Позиционирование трёх продуктов

Одна кодовая база агентов (`agency_agents` + рантайм из `marketing-office`) → три формы доставки. Каждая закрывает **разную боль разной аудитории**, поэтому каннибализация управляема.

| Параметр | **A. SaaS NEXUS** | **B. Marketplace-плагин** | **C. Self-hosted** |
|---|---|---|---|
| Слоган | «Команда из 150 AI-специалистов в браузере, без своего ключа Anthropic» | «Профи-агенты прямо в твоём Claude Code» | «NEXUS на твоём сервере, твои данные, твой ключ» |
| Кто | SMB, маркетологи, фрилансеры, малые агентства РФ; не-технари | Разработчики, тех-команды, инди-хакеры, уже в Claude Code | Средние/крупные компании, агентства, госсектор, требования к данным |
| Боль | «Нужен контент/стратегия под Рунет, нет своих экспертов и нет API-ключа» | «Claude Code мощный, но нет готовых ролей/промптов под маркетинг и спецзадачи» | «Нельзя гонять данные через чужой SaaS; нужен контроль, бренд, кастом» |
| Что покупает | Доступ к web-чату с агентами + готовый рантайм (мы платим за токены) | Набор `.md`-агентов + апдейты библиотеки (клиент платит за токены сам) | Инстанс + поддержка + обновления библиотеки + SLA |
| Кто платит за токены | **МЫ** (зашито в подписку, главный драйвер себестоимости) | **КЛИЕНТ** (свой Anthropic-ключ) | **КЛИЕНТ** (свой ключ/сервер) |
| Ценовой якорь | Подписка $19–199/мес | $9–49/мес или $99–299 разово | $300–3000/мес абонплата + setup |
| Маржа | Средняя (токены съедают) | **Высокая** (нет наших токенов) | Высокая (сервис, не токены) |
| Скорость до выручки | Средняя (нужен биллинг+auth+учёт токенов) | **Быстрая** (упаковка готовых .md + Gumroad/маркетплейс) | Медленная (продажи, договоры, внедрение) |

### Как разводим каннибализацию
- **Разные точки потребления.** SaaS — браузер, ключ наш. Плагин — IDE/Claude Code, ключ клиента. Self-hosted — сервер клиента. Пользователь почти никогда не «и там и там».
- **Разный профиль платёжеспособности.** Не-технарь не поставит плагин в Claude Code; разработчику не нужен наш web-чат. Пересечение <10% (оценка).
- **Ценовая лестница, а не конкуренция.** Плагин — «вход в экосистему» (дёшево, вирусится). SaaS — «не хочу возиться с ключом». Self-hosted — «не хочу зависеть от вас». Это апселл-путь: плагин → SaaS Team → Self-hosted Enterprise.
- **Контент-дифференциация.** В плагин даём агентов, но НЕ даём наш курируемый рантайм, оркестрацию, дашборды учёта, готовые «команды по департаментам» — это эксклюзив SaaS/Enterprise.

### Линия позиционирования (одной строкой)
- **A. SaaS NEXUS** — «AI-маркетинговая команда под Рунет в браузере, без своего API-ключа».
- **B. Плагин** — «150+ готовых профи-агентов в Claude Code за цену одного обеда».
- **C. Self-hosted** — «То же NEXUS, но внутри твоего периметра, с поддержкой и SLA».

## 3. Unit-экономика

> Ставки Anthropic Claude **Sonnet 4.6**: **$3 / 1M входных**, **$15 / 1M выходных токенов** (оценка, проверить актуальность). Текущий код: `MAX_TOKENS=2048` на ответ, без истории диалога в одном запросе по умолчанию.

### 3.1. Стоимость типового диалога в токенах

Системный промпт агента (body `.md`) — обычно объёмный. Считаем **консервативно** (с запасом):

| Компонент | Токенов | Тип | Цена за 1M | Стоимость |
|---|---|---|---|---|
| System prompt агента | ~2 500 | input | $3 | $0.0075 |
| Запрос пользователя | ~500 | input | $3 | $0.0015 |
| Ответ агента (MAX_TOKENS) | ~2 000 | output | $15 | $0.0300 |
| **Итого 1 диалог** | ~5 000 | | | **≈ $0.039** |

**Стоимость одного типового диалога ≈ $0.04 ≈ 3.5₽** (оценка).

Формула:
`cost = (input_tokens/1e6)*3 + (output_tokens/1e6)*15`
= (3000/1e6)*3 + (2000/1e6)*15 = 0.009 + 0.030 = **$0.039**.

Если включить историю (контекст растёт) — input может удвоиться-утроиться → закладываем **средний реальный диалог $0.05–0.08** для тяжёлых юзеров. Берём **$0.05 (≈4.5₽)** как плановую среднюю себестоимость диалога для SaaS.

### 3.2. Себестоимость активного юзера (только SaaS — там токены наши)

| План | Лимит сообщ./мес | Себест. токенов при 100% выборке ($0.05/диалог) | Реальная выборка* | Реальная себест. токенов |
|---|---|---|---|---|
| Free | 50 | $2.5 | 60% | $1.5 |
| Pro | 1 000 | $50 | 25% (≈250) | **$12.5** |
| Team | 5 000 (пул) | $250 | 30% (≈1500) | **$75** |
| Enterprise | 20 000 | $1 000 | 35% | $350 |

*Реальная выборка — % лимита, который средний юзер реально использует (большинство не выбирают лимит; это и есть наша защита от убытка).

Доп. косты на юзера (оценка): инфра/хостинг ~$1–2, платёжная комиссия ~3% (Stripe/ЮKassa), саппорт амортизированно ~$1.

### 3.3. Маржа по тарифам и порог окупаемости (SaaS)

| План | Выручка/мес | Токены (реал.) | Комиссия 3% | Инфра+саппорт | **Валовая маржа** | **% маржи** |
|---|---|---|---|---|---|---|
| Pro $19 | $19 | $12.5 | $0.6 | $2.5 | **$3.4** | **~18%** |
| Pro (низкая выборка 15%) | $19 | $7.5 | $0.6 | $2.5 | $8.4 | ~44% |
| Team $79 | $79 | $75 | $2.4 | $5 | **−$3.4** | **УБЫТОК** |
| Team (выборка 20%) | $79 | $50 | $2.4 | $5 | $21.6 | ~27% |
| Enterprise $299 | $299 | $350 | $9 | $10 | **−$70** | **УБЫТОК при кастом-лимитах** |

**Ключевые выводы по unit-экономике:**
1. **Pro окупается** на средней выборке (~18–44% маржи). Главный риск — «тяжёлые» юзеры, выбирающие весь лимит. Защита: лимит 1000 рассчитан так, что 100%-выборка ($50) почти равна цене — поэтому **обязателен овердрафт-биллинг / hard-cap**, иначе единичные тяжёлые юзеры съедают прибыль.
2. **Team на максимуме лимита убыточен** — лимит 5000 при 100% выборке = $250 > $79. **Нужно либо снизить лимит до ~1500–2000, либо ввести usage-based доплату.** Рекомендую: пул 2 000 сообщ. в базе + докупка пакетов.
3. **Enterprise НЕ продавать по фиксу с большим лимитом** — только **cost-plus**: базовая плата за платформу/саппорт + токены по счётчику с наценкой 30–40%.
4. **Порог окупаемости платформы (оценка):** при марже ~$5/Pro-юзер и постоянных издержках разработки/инфры ~$3–5к/мес → нужно **~700–1000 платящих Pro** или эквивалент. Плагин и self-hosted дают высокую маржу и **закрывают порог быстрее** (см. §5).
5. **Главный рычаг прибыли SaaS:** удешевить токены — кэш системных промптов (Anthropic prompt caching даёт до −90% на повторный input), урезать `MAX_TOKENS` для лёгких задач, роутинг на Haiku для простых агентов. Prompt caching системного промпта снижает себестоимость диалога с $0.05 до ~$0.02–0.03 → маржа Pro растёт до 40–60%.

## 4. Биллинг-стратегия

**Двухконтурная оплата** (геофактор: РФ-карты не проходят Stripe, мировые — не всегда ЮKassa):
- **Мир / валюта:** Stripe (подписки, usage-based, customer portal) или Paddle/LemonSqueezy как Merchant-of-Record (снимает headache с налогами/VAT для глобала).
- **РФ / рубли:** ЮKassa (рекуррент, СБП, карты МИР) либо CloudPayments как fallback. Для self-hosted и Enterprise РФ — счёт/договор (b2b безнал).

| Продукт | Модель биллинга | Обоснование |
|---|---|---|
| **A. SaaS** | **Гибрид: seat-base подписка + usage-overdraft + credits** | Подписка = предсказуемая выручка и простой онбординг. Но токены — переменный кост, поэтому поверх подписки нужен usage-cap и докупка пакетов («кредиты сообщений»). Чистый seat-based опасен (тяжёлый юзер = убыток), чистый usage пугает не-технарей. Гибрид балансирует. |
| **B. Плагин** | **Seat-based подписка + Lifetime опция** | Токены на клиенте → переменных костов у нас нет, usage-billing не нужен. Простая фикс-подписка/seat максимизирует маржу и понятна. Lifetime — для разовой выручки и снятия барьера. |
| **C. Self-hosted** | **Setup-fee + фиксированная абонплата (seat-tiers) + cost-plus на Enterprise** | Это сервисный контракт, не SaaS. Setup окупает внедрение, абонплата = поддержка/SLA/обновления. Usage не считаем — токены у клиента. На самом верху — cost-plus за кастом-разработку агентов. |

**Технически для SaaS обязательно (чего сейчас НЕТ в коде):**
1. **Auth** (email-magic-link или OAuth) + **БД** (Postgres: users, subscriptions, usage).
2. **Учёт токенов на каждый `/api/chat`** — писать `input_tokens/output_tokens` из ответа SDK в usage-таблицу. Это критично и для биллинга, и для защиты от убытка.
3. **Мидлвар лимитов**: проверка остатка сообщений/кредитов перед вызовом Anthropic, hard-cap при превышении.
4. **Вебхуки** Stripe/ЮKassa → апдейт статуса подписки.
5. **Prompt caching** системных промптов агентов (прямое снижение себестоимости, см. §3.3).

## 5. MVP scope и приоритет (Roadmap по фазам)

### Что выкатываем ПЕРВЫМ и почему

**ПЕРВЫЙ = B. Marketplace-плагин.** Однозначно.

Обоснование (через скорость-до-выручки и текущий код):
1. **Минимум разработки.** Агенты уже в виде `.md`. Продукт = установщик + лендинг + платёжка (Gumroad/LemonSqueezy/ЮKassa). НЕ нужны auth, БД, мультитенантность, учёт токенов, рантайм-инфра. Это **дни-недели**, а не месяцы.
2. **Высокая маржа сразу** — токены платит клиент, наш COGS почти ноль. Каждая продажа = почти чистая прибыль → быстрее закрывает порог окупаемости платформы.
3. **Нулевой инфра-риск.** Нет наших токенов = нет риска убытка от тяжёлых юзеров.
4. **Маркетинговый плацдарм.** Дешёвый виральный продукт ($9–29) собирает аудиторию и соц-доказательство, которое потом конвертит в SaaS и Enterprise.

**SaaS (A) — ВТОРОЙ**, потому что даёт recurring и широкую РФ-аудиторию не-технарей, но требует построить auth+БД+биллинг+учёт токенов+лимиты (см. §4). Это 4–8 недель и финансовый риск без учёта токенов.

**Self-hosted (C) — ТРЕТИЙ**, sales-driven, появляется по входящим запросам Enterprise. Не строим спекулятивно — упаковываем под первого реального клиента.

### Roadmap по фазам

| Фаза | Срок (оценка) | Продукт | Scope | Метрика выхода |
|---|---|---|---|---|
| **Ф0. Упаковка плагина** | 1–2 нед | B | Установщик `.md`→`~/.claude/agents/`, лендинг, оплата (LemonSqueezy + ЮKassa), 3 тарифа | Первые 10 платящих |
| **Ф1. Рост плагина** | +2–4 нед | B | Free-tier виральность, апдейт-механизм, Team-license, отзывы | 100 платящих / MRR ≥ $1.5к |
| **Ф2. SaaS MVP** | 4–8 нед | A | Auth + Postgres + Stripe/ЮKassa + учёт токенов + лимиты + prompt caching, на базе `marketing-office` | 50 платящих Pro |
| **Ф3. SaaS scale** | +4–6 нед | A | Team-план, пул-сообщений, дашборд usage, годовая оплата, овердрафт | 700–1000 Pro (порог окупаемости) |
| **Ф4. Self-hosted** | по запросу | C | Docker-инсталл, SSO, runbook саппорта, договор/SLA | Первый Enterprise-контракт |

### MVP-scope SaaS (минимум для Ф2, чего нет в коде сейчас)
- [ ] Postgres: `users`, `subscriptions`, `usage_events(user_id, agent, in_tok, out_tok, cost, ts)`
- [ ] Auth (magic-link), сессии
- [ ] Мидлвар: проверка лимита перед `/api/chat`, запись токенов после
- [ ] Stripe Checkout + ЮKassa, вебхуки статуса подписки
- [ ] Prompt caching системных промптов (снижение COGS)
- [ ] Гейтинг агентов/департаментов по тарифу
- **Non-goals v1:** мобильное приложение, командный shared-workspace, API для интеграций, white-label (отдать в Self-hosted).

## 6. Go-to-market (приоритетный продукт — B. Плагин)

**Цель Ф0–Ф1:** 100 платящих / MRR ≥ $1.5к за ~6 недель.

**Первые каналы (в порядке отдачи):**
